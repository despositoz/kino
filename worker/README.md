# Gemini для Кинодневника

Worker нужен, чтобы ключ Gemini не попадал в публичный код GitHub Pages.

## Подключение

1. Создай новый API-ключ в Google AI Studio. Старый ключ из переписки использовать нельзя.
2. В терминале открой эту папку и выполни `npx wrangler login`.
3. Сохрани ключ командой `npx wrangler secret put GEMINI_API_KEY`.
4. Опубликуй Worker командой `npx wrangler deploy`.
5. Скопируй полученный адрес и укажи два маршрута в `webapp/index.html`:
   `/review` — в meta-параметре `kino-ai-endpoint`, `/feed` — в
   `kino-ai-feed-endpoint`.

Worker обслуживает оба сценария: создание записи (`POST /review`) и карточку
«А ты знал?» для Ленты (`POST /feed`). Для факта клиент сначала получает из
TMDB точные данные о фильме — режиссёра, бюджет, мировые сборы и связанные
работы, — а Gemini только выбирает и формулирует один факт. Сырые заметки в
маршрут `/feed` не отправляются.

Ключ нельзя записывать в `worker.js`, `wrangler.jsonc`, `.env`, `webapp/` или отправлять в чат.
